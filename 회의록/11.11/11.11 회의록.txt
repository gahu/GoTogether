11.11(토) 회의내용 정리
●회의 내용
프론트 UI 마무리
 - 모든 fragment 전환 작동 확인 
 - 화면 전환 마무리 -> roadMap형식으로 어떻게 작동하고 fragment가 이동하는지 그릴 수 있도록
 - 디자인 검토해서 보완 및 추가사항 연결
 - 폰트 변경
 - 코스/ 지도에서 주요지점, 안전지점 확인해서 정확한 위치 표시 및 불필요한 부분 제거
 - 오픈소스 확인해서 해당 정보 확인

백엔드
 - GPS 본인이 원할때만 작동되도록 끄기/켜기 만들기
 - 네트워크 연결 방식 연구 (Public class ProcessNetwork CourseBaseInfo Thread extends AsyncTask<void, void, String>
 - 스탬프 자동인식 방법에 관련해서 의견
 - 사용자 정보 저장방식(스탬프를 찍으면 어떤식으로 저장되게 할것인가)
 - 커뮤니티 관련 저장소를 어떻게 할 것인가?(서버문제)

해결해야할 사항
 - community layout 재구성
 - FnQ는 소스보고 순서 최적화 -> 기계별 작동 확인
 - 백버튼 메인에서 토스트알림 뜨도록 변경
 - fragment가 남아있는 부분(기타, cafe 등)
 - 화면 scroll -> 전체를 스크롤해서 중간 fragment를 위로 올릴것인가 / 윗부분을 keydown 방식으로 임시적으로 사라지게 할것인가
 
디자인부분
 - 탭부분 background Img
 - Title background Img
 - 메인 큰이미지, 뒤 배경없는 버튼이미지
 - 코스이미지(스탬프/번호)
 - 메인 -> 메인 이미지를 어떻게 할 것인가

●개선방안
 - 메인 화면 -> 메인을 이미지화하고 버튼을 위에 두어서 작동하도록 만듬
 - FnQ -> 기본 방식 그대로 가되, 문의하기(관리자와 연결) & 기본질문 같은걸 개별 작업
 - 백버튼 -> 메인 메뉴에서만 토스트화면 뜨도록하고 다른 fragment는 메인으로 돌아가게 하기
 - 스탬프 -> 스탬프 자동 등록을 강점으로 세우고 또 다른 스탬프를 모으는 방법 모색\
 - 커뮤니티 -> 커뮤니티 저장 장소를 Google Cloud로 하고, 자신의 기기에 따라서 닉네임을 작성하여 커뮤니티 사용하도록 하기

●개인분담 작업
 - 재오 : 코스/스탬프가 정상적으로 작동되도록 MenuActivity.class에 적용 / network작동 원리 분석하고 최적화 / 화면전환 완성
 - 관현 : GPS 켜고 끄는 작동 만들기 / 백버튼 다른 fragment에서는 메인으로 돌아가도록, 메인에서는 토스트창 띄우기
 - 지호 : community layout 재구성해서 완성 / 커뮤니티 관련 저장소 Google Cloud를 사용해서 만들기 / 커뮤니티 닉네임, 스탬프 저장방식 연구
 - 윤규 : 이미지 작업 완료 및 전달
